# Project 1 — Database Design & Requirements Analysis
**Course: INFO 579 | University of Arizona**

## 📌 Objective
Design a complete database solution for a Restaurant Management System
from scratch — starting with business requirements analysis through
to conceptual and physical data models.

## 🔑 Key Deliverables

### Requirements Analysis
Documented business rules for a restaurant information system covering:
- Menu item management (name, price, category, stock)
- Customer reservation system (date, time, guests, table assignment)
- Order management (multiple items per order, status tracking)
- Customer feedback system (rating + comment)
- Membership program (Gold/Silver/Bronze tiers)
- Table management (number, capacity)

### Conceptual Data Model (CDM)
Designed using **Crow's Foot notation** — visually representing all
entities, relationships, and cardinalities.

### Physical Data Model (PDM)
Translated CDM into implementable schema with primary keys, foreign
keys, data types, and constraints.
